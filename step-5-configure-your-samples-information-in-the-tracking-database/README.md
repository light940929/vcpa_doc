# Step 5: Configure your samples information in the tracking database

**Before users can schedule / submit jobs, users need to add the the sample information via the VCPA Sample Management Application Programming Interfaces \(API\) to the tracking database. This step is necessary so that users can manage the jobs running status as well as viewing the quality metrics generated by the pipeline. Here's what you do for each sample.**

[**5.1 Input the sample information to the tracking database**](https://niagads.gitbook.io/vcpa/step-5-configure-your-samples-information-in-the-tracking-database/5.1-input-the-sample-information-to-the-tracking-database)

[**5.3 Populate the tracking database with the designated result folder for each sample to be processed**](https://niagads.gitbook.io/vcpa/step-5-configure-your-samples-information-in-the-tracking-database/5.3-populate-the-tracking-database-with-the-designated-result-folder-for-each-sample-to-be-processed)

[**5.4 Input PCR protocol information into the tracking database**](https://niagads.gitbook.io/vcpa/step-5-configure-your-samples-information-in-the-tracking-database/5.4-input-pcr-protocol-information-into-the-tracking-database)
